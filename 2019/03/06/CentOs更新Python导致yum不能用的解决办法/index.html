Mar 6, 2019 | Linux | 阅读

最近在搞一个微信机器人的demo，运行在阿里云上，项目需要python环境的版本是2.7.X，然而服务器默认依赖的版本仍然是2.4.X，所以更新了python的版本（更新后ython版本是2.7.X），运行

但是项目依旧无法运行，于是百度，

更改项目文件对python依赖的连接

问题没有解决。

继续百度，搜到一个解决方案：

更改系统依赖python的软连接,然后把项目依赖从python2.7改回python

但是问题仍然没有解决。

反而导致yum不能用了。

干脆完全卸载python和yum

重新安装！
1.完全卸载python
2.完全卸载yum
