使用Spring Boot aop和自定义异常类统一处理接口异常

Aug 23, 2019 | Spring Boot | 阅读
问题：

老项目之前没有做统一异常处理，接口数量比较多，大都是捕获异常后打印日志，加上项目的实例比较多，一出现异常，就需要到众多实例中找日志信息。虽然运维系统会根据日志类型log_info或log_error进行分类，并且会根据时间切片，但是由于日志数量多，定位异常还是很困难。

需求：

针对多种不同类型的异常信息统一捕获并处理，同时定位报错接口的位置，记录异常信息，接口入参，错误代码，存入数据库。再启动一个异步定时任务，当异常数量到达一定数量时将异常信息通过短信或邮件发送给管理员。

采用方案：
使用spring 的 aop 技术切到自定义注解上，针对不同注解标志进行参数解析，记录日志
