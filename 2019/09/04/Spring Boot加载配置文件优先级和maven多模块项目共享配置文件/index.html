Spring Boot加载配置文件优先级和maven多模块项目共享配置文件

Sep 4, 2019 | Spring BootMaven | 阅读
现在是什么情况？

现有Maven构建的Spring Boot多模块项目(所有模块都是平级的)，每个模块实现一块功能,其他模块引用base模块中的方法，但每个模块有单独数属性的配置文件，且部分模块有多配置文件以适应不同的生产环境。模块中配置文件格式不统一,既有.properties也有.yml。

出现了什么问题？

web service模块调用base模块的方法，在base模块中获取application.yml的sysuri属性时属性为Null

检查了web service模块的POM文件

尝试调用其它base模块方法，发现其它方法调用没问题，但所有base模块的application.yml属性的方法都获取不到。属性注入都为空。

尝试解决问题
  1. 先检查application.yml文件的格式是否错误？
格式没什么问题。
  1. 考虑是否是properties,yml的优先级加载顺序不同，造成了配置覆盖？
    在spring-boot-starter-parent里可以看到

当两个文件中有配置信息冲突时,优先级是:properties>yml,也就是properties里配置的内容会覆盖另外一个的配置。

web service模块的配置文件中并没有相同属性名，可以排除。

  1. 仔细检查base模块的POM文件看看
果然，POM文件中未配置将resource下yml文件打包。再反编译打好的jar包，发现确实没有.yam结尾的配置文件。

然后install一下base。
再检查一下打好的jar包，正常了。

此时再从web service调用base模块，一切正常了。
